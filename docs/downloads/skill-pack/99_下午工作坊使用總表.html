Skill Pack

下午工作坊使用總表

下午目標

早上讓老師親自體驗《我們的島》的教育力量。

下午則協助老師完成五件事：

  1. 找到一個真實可用的教學場景。
  2. 練習接住可能出現的學生反應。
  3. 看見學生、師長與制度中可能出現的歧視情境。
  4. 辨識歧視事件中的情緒與後續副作用。
  5. 寫下第一次推廣的低門檻承諾。

建議流程

13:30-13:45 從玩家到教師

目標

讓老師把早上的玩家經驗轉成教學設計視角。

引導語

早上你作為玩家，經歷了《我們的島》。下午我們要換一個位置：如果你的學生也經歷這套遊戲，你要如何讓這份體驗變成安全而有意義的學習？

討論題

  1. 早上哪一刻讓你感覺這套遊戲有教育力量？
  2. 如果換成你的學生，這一刻可能產生什麼學習？
  3. 如果換成你的學生，這一刻也可能產生什麼風險？

13:45-14:25 真實場景設計

使用文件

  • 01_三種推廣版本.md
  • 02_一頁式教學設計單.md
  • 07_校園歧視情境分類清單.md

任務

每位老師選一個自己真的可能使用的場域，完成一頁式教學設計單。

引導提醒

請不要設計完美教案。請設計一個三週內有可能真的試一次的版本。

小組分享

每人用 2 分鐘分享：

  1. 我要帶的對象是誰？
  2. 我選擇哪一種版本？
  3. 我最擔心的風險是什麼？
  4. 我會先做哪一個調整？

14:25-14:40 校園情境辨識

使用文件

  • 07_校園歧視情境分類清單.md

任務

每組從清單中選 2 類最貼近自己場域的歧視情境，討論：

  1. 這類情境在我的學校可能怎麼出現？
  2. 它比較可能發生在學生之間、師生之間，還是制度安排裡？
  3. 如果我要帶《我們的島》，需要先注意什麼安全設計？

引導提醒

這段不是要指認誰做錯，而是幫老師把遊戲隱喻連回校園現場。請把語氣放在辨識模式，而不是審判模式。

14:40-15:15 敏感情境演練

使用文件

  • 03_開場與安全聲明話術.md
  • 04_敏感情境回應卡.md
  • 08_師長版情境卡.md
  • 10_歧視事件情緒地圖.md
  • 11_事件後副作用與修復檢核.md

任務

每組抽 1 張學生版情境卡，或 1 張師長版情境卡，練習老師如何回應。

演練步驟

  1. 讀出情境。
  2. 討論學生背後可能的需要或狀態。
  3. 設計老師回應。
  4. 用三步驟練習：先接住、再打開、帶回學習。
  5. 如果抽到師長版情境卡，再補一題：如果老師已經說錯或處理不周，可以怎麼修復？
  6. 補看情緒地圖：被影響者、造成傷害者、旁觀者可能各有什麼情緒？
  7. 補看副作用檢核：如果沒有處理好，可能往哪裡擴散？
  8. 全場示範 1 分鐘。

回饋方式

其他組只回饋兩件事：

  1. 哪一句話讓學生比較可能被接住？
  2. 哪裡可以更清楚地帶回學習？
  3. 是否有看見事件後需要追蹤的副作用？

15:15-15:25 休息

15:25-15:50 同儕諮詢

目標

讓老師把自己的設計變得更可行。

三人一組流程

  1. 提案老師說明自己的設計與卡點，2 分鐘。
  2. 夥伴只問釐清問題，2 分鐘。
  3. 提案老師轉身或安靜聽，夥伴討論建議，5 分鐘。
  4. 提案老師回應：我聽到最有幫助的是什麼，2 分鐘。
  5. 交換角色。

建議聚焦

  • 如何降低第一次推廣門檻？
  • 哪一段需要安全設計？
  • 哪一題反思最適合這群學生？
  • 是否需要輔導老師、導師或行政協助？
  • 如果發生真實情緒或歧視事件，後續如何追蹤？

15:50-16:10 反思題與推廣版本定稿

使用文件

  • 05_ORID反思題庫.md

任務

每位老師完成：

  1. 選定輕量、標準或延伸版本。
  2. 寫下 O/R/I/D 各一題。
  3. 寫下一句自己的開場安全聲明。

引導提醒

遊戲後反思不要貪多。各層次一題即可，重點是讓學生從事件、感受、理解走到下一步。

16:10-16:30 第一步承諾與支持需求

使用文件

  • 06_第一步承諾卡與推廣回報.md

任務

每位老師完成第一步承諾卡。

分享句型

我願意先嘗試的版本是...

我會帶的對象是...

我最擔心的是...

我下一步會先做...

我需要的支持是...

下午結語

你不需要第一次就把《我們的島》帶得完美。

你只需要完成一次真實而安全的嘗試，讓學生有機會看見：一句話、一個選擇、一段關係，都可能讓橋斷裂，也可能讓橋重新連起來。

引導師檢核

下午結束前，確認每位老師至少帶走：

  • 一個真實可用的教學場景
  • 一種推廣版本
  • 一句開場安全聲明
  • 一組 ORID 反思問題
  • 一個風險處理提醒
  • 一個情緒判讀提醒
  • 一個事件後修復提醒
  • 一張第一步承諾卡
